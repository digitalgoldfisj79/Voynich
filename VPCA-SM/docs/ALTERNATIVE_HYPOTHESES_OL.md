### Avoiding Confirmation Bias

**Critical safeguards:**
1. Test ALL plausible hypotheses, not just favorite
2. Design tests that could reject leading hypothesis
3. Update confidence DOWN if tests fail
4. Consider null results seriously
5. Don't "explain away" negative results

**Example:**
If TEST 2.2 shows NO dry enrichment:
- âœ… Hypothesis A confidence goes DOWN (not sideways)
- âœ… Move to next hypothesis
- âŒ Don't adjust predictions to fit data
- âŒ Don't dismiss test as "unclear"
