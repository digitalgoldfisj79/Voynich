## üî¨ EPISTEMIC FRAMEWORK (Updated)

### Confidence Levels Explained

**Structural Confidence** (What we measure directly):
- Based on: VPCA clustering, distribution, behavioral consistency
- High (70-75%): Strong evidence across multiple measures
- Medium (50-70%): Good evidence but some uncertainty
- Low (<50%): Weak or conflicting evidence

**Semantic Confidence** (What we want but rarely have):
- Based on: Actual meaning assignments validated by testing
- High (‚â•40%): Multiple successful tests
- Medium (20-40%): Some test support
- Low (<20%): No test support or hypothesis rejected

**CRITICAL RULE:**
> **Never claim semantic confidence based only on structural evidence**
> 
> VPCA tells us word CLASSES (adjective, verb), not MEANINGS (hot, run)

## üí° KEY LESSONS FROM VALIDATION

### 1. Structure ‚â† Semantics

- VPCA gives CATEGORY (quality vs process vs relation)
- Testing gives CONTENT (which quality, which process)
- Must validate separately

### 2. Medieval Parallels Are Suggestive Only

- Etymological connections are hypotheses, not evidence
- Need systematic corpus comparison
- Cherry-picking examples is not proof

### 3. Multiple Tests Required

- Single test can give false positives/negatives
- Need converging evidence from multiple independent tests
- Target: 3-5 tests per hypothesis for publication

### 4. Negative Results Are Valuable

- Failed tests eliminate hypotheses ‚úÖ
- Narrow search space ‚úÖ
- Guide future research ‚úÖ

### 5. Confidence Must Match Evidence

- Speculation deserves low confidence (<20%)
- Validated hypotheses earn higher confidence (40-70%)
- Never conflate "could be" with "probably is"
